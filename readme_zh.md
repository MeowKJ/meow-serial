## 这是干什么的

喵喵的串口工具是一个基于浏览器的串口 / WebSocket 调试工具，主要分成三个使用区域：

- `画布`：拖拽控件搭实时面板
- `终端`：查看收发数据、发送命令
- `协议`：把原始数据解析成有名字的通道

这个项目更偏“使用型工具”而不是“展示型项目”。核心目标是让你尽快连上设备、确认数据、做出看板，并把整套配置保存下来。

## 使用前准备

- Node.js 18+
- 如果要用 Web Serial，建议使用 Chromium 内核浏览器
- 如果设备走网络，也可以使用 WebSocket 方式接入

## 安装与启动

### 安装依赖

```bash
pnpm install
```

或者：

```bash
npm install
```

### 启动项目

```bash
pnpm dev
```

启动后打开终端里显示的本地 Vite 地址即可。

## 如何使用

### 1. 添加并配置端口

在左侧边栏里可以：

- 添加串口或 WebSocket 端口
- 设置波特率和传输参数
- 选择设备
- 连接或断开

如果只是先确认设备有没有在说话，建议先连上再看终端，不要一开始就急着配图表。

### 2. 在协议页选择解析方式

如果你的数据不是单纯文本，而是结构化字段，去 `协议` 页处理。

这里可以：

- 保持 `raw`，用于最直接的原始调试
- 选择内置解析器
- 新建或编辑协议配置
- 把解析结果映射成具名通道

一旦解析器生效，通道就能被波形图、数值卡片、仪表盘等控件直接消费。

### 3. 在终端页确认收发数据

`终端` 页适合做第一轮确认：

- 看 RX / TX 日志
- 切换显示格式
- 发送 UTF-8 或 HEX 命令
- 使用 `CR` / `LF`
- 需要时导出日志

建议先在这里确认“设备真的在发”“我真的能发回去”，再去搭画布。

### 4. 在画布页搭控件面板

点击顶部 `控件` 按钮，把控件加到画布里。当前适合常用调试场景的控件包括：

- 波形图
- 迷你波形图
- 数值显示
- 仪表盘
- FFT
- 直方图
- XY 图
- 按钮
- 滑块
- 触发器
- 迷你终端

控件支持拖动、缩放、复制、图层调整和右侧设置面板配置。

### 5. 配置控制类控件

控制类控件不是摆设，尤其是按钮控件，现在已经补得更完整了。

按钮控件支持：

- 更合理的默认大小，直接点更顺手
- 设置按钮文字和发送命令
- 选择按钮风格
- 选择 UTF-8 / HEX 发送格式
- 配置 `CR` / `LF`

新增按钮控件时，会自动打开右侧设置面板，方便立即配置，不用再多点一次。

### 6. 保存全局工作区

顶部栏支持导出 / 导入全局工作区 JSON。当前会一起保存：

- 控件与布局
- 串口与解析器选择
- 主题设置
- 画布背景模式
- 界面语言

如果你经常在不同设备上调同一套协议，或者想给同事复用一份配置，这一项会很实用。

## 推荐使用流程

大多数情况下，建议按这个顺序来：

1. 先连接端口
2. 在 `终端` 页确认数据有没有正常进来
3. 在 `协议` 页选择或配置解析器
4. 确认通道值是否正常更新
5. 回到 `画布` 页搭控件
6. 最后导出工作区 JSON 备份

## 多语言支持

项目已内置轻量 i18n：

- 支持 `zh-CN` 和 `en`
- 可以在右上角切换语言
- 语言会保存在本地
- 语言也会写入全局工作区 JSON

当前优先覆盖了应用外壳、控件面板和关键配置流程，后续还可以继续补更多页面。

## 开发方式

开发部分保持尽量简单，适合边改边看。

### 常用命令

```bash
pnpm dev
pnpm build
pnpm preview
```

### 主要目录

```text
src/
├── components/   # 页面壳层和主视图
├── widgets/      # 画布控件
├── stores/       # Pinia 状态管理
├── parsers/      # 协议解析器
├── utils/        # 串口、存储、导出等工具
├── i18n/         # 多语言消息与辅助函数
└── styles/       # 全局样式
```

如果要扩展项目，常见入口大概是：

- `src/components/`：改页面和工具栏
- `src/widgets/`：新增或修改控件
- `src/parsers/`：补协议支持
- `src/i18n/`：补翻译内容
